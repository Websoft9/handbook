# 发布交付

## 做什么？

产品发布是供应链的终极目标：把产品送到客户手中。

对于我们的业务来说就是：将产品发布到云平台或客户的云服务器中。


## 怎么做？

### 发布{#pulish}

发布到云平台是销售产品的前置工作，它由商品上传到仓库和商品信息发布两个部分工作组成。

其中，信息录入是关键，需重点关注商品的可购买性和合规性，同时关注ID不可更改性、所属分类、亮点提炼等

#### 流程

上架采用“两人分工，一提一审”的方案。假定两人的角色名称分别为：上架专员和主管

> 虽然发布之后，销售还需要对商品信息进行更优化的描述，但发布工作仍然尽量做得更好。

#### 录入

根据：[商品表现力模型](../marketplace/offer.md)录入商品信息。  

与此同时，商品**定价**需要进行磋商。  

#### 内审

内部审核录入的内容

#### 提交

提交云市场平台审核以及处理过程中的问题

#### 后置工作

成功上架后还需要完成：购买测试、删除镜像和快照等工作


### 交付

交付到客户的服务器可归类于售后工作
