# 发布交付

## 做什么？

产品发布是供应链的终极目标：把产品送到客户手中。

对于我们的业务来说就是：将产品发布到云平台或客户的云服务器中。


## 怎么做？

### 发布{#pulish}

发布到云平台是销售产品的前置工作，它由镜像推送到仓库(镜像复制到不同销售区域)和商品发布两个部分工作组成。  

发布需要遵循的原则：

1. 采用“两人分工，一提一审”的方案。假定两人的角色名称分别为：上架专员和主管
2. 发布之后，还需要对商品信息进行更优化的描述，尽量商品可被检索

#### 镜像检查

部分云平台在商品发布之前还有镜像检查的前置动作：

- **AWS**：将镜像推送到资产库，等待 AWS 安全检测并返回结果

- **华为云**：将镜像推送到资产库，等待华为云安全检测并返回结果

- **HUAWEI CLOUD**: 

  1. 将镜像推送到资产库，人工做安全扫描（漏洞扫描+文件病毒扫描），并向华为云相关人员邮件发送扫描报告，待他们审核。安全扫描过程：

  2. 漏洞扫描：登录华为云 HSS 控制台（"安全与合规"->"主机安全服务HSS"），对服务器进行安全检查：
      a. 获取 agent 安装脚本，在测试 ECS 服务器上安装agent（需等待几分钟才生效）  
      b. 对已安装 agent 的 ECS 开启防护，根据指引**按量购买**的HSS服务企业版，购买完成后等待自动检测的结果    
      c. 将风险预防->基线检测的页面截图保存    
      d. 将漏洞管理->"主机视图"->导出PDF的报告保存    

  3. 病毒扫描：登录华为云 ECS 控制台，连接到服务器，先后运行卡巴斯基的文件病毒扫描和产看病毒隔离文件命令（ECS 服务器**基本信息**标签页截图；华为云远程连接至服务器，终端运行 卡巴斯基扫描结果 截图）
     ```
     docker run -it --rm -v '/':/data tabledevil/kaspersky scan
     docker run -it --rm -v '/':/data tabledevil/kaspersky query
     ```

#### 商品发布

商品信息录入是商品发布的键，需重点关注商品的可购买性、定价和合规性，同时关注ID不可更改性、所属分类、亮点提炼、关键字设置等。   

具体参考：[商品表现力模型](../sales/marketplace/offer) 

与此同时，商品**定价**需要进行多人磋商。  

#### 内审

内部审核商品信息

#### 提交

提交云市场平台审核以及处理过程中的问题

#### 后置工作

成功上架后还需要完成：购买测试、删除镜像和快照等工作

### 交付

交付到客户的服务器可归类于售后工作
