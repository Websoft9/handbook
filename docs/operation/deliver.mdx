# 发布交付

## 做什么？

产品发布是供应链的终极目标：把产品送到客户手中。

对于我们的业务来说就是：将产品发布到云平台或客户的云服务器中。


## 怎么做？

### 发布{#pulish}

发布到云平台是销售产品的前置工作，它由商品上传到仓库和商品信息发布两个部分工作组成。  

发布需要遵循的原则：

1. 采用“两人分工，一提一审”的方案。假定两人的角色名称分别为：上架专员和主管
2. 发布之后，还需要对商品信息进行更优化的描述，尽量商品可被检索

#### 镜像检查

部分云平台在录入信息之前还有镜像检查的前置动作：

- **AWS**：将镜像推送到资产库，等待 AWS 安全检测并返回结果

- **华为云**：将镜像推送到资产库，等待华为云安全检测并返回结果

- **HUAWEI CLOUD**: 

  1. 将镜像推送到资产库，等待安全检测并返回结果（此项任务通过的概率相对较低，故先做）
  2. 登录华为云 HSS 控制台，对服务器进行安全检查（基线检测的结果截图，漏洞扫描的结果 pdf 导出）
  3. 登录华为云 ECS 控制台，连接到服务器，运行卡巴斯基的文件病毒扫描命令（ECS 服务器**基本信息**标签页截图，终端运行 卡巴斯基扫描结果 截图）
     ```
     docker run -it --rm -v '/':/data tabledevil/kaspersky scan
     ```

#### 信息录入

商品信息录入是关键，需重点关注商品的可购买性和合规性，同时关注ID不可更改性、所属分类、亮点提炼等。   

具体参考：[商品表现力模型](../marketplace/offer) 

与此同时，商品**定价**需要进行多人磋商。
